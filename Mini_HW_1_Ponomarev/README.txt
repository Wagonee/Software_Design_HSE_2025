Основные компоненты системы: 
Animal - абстрактный класс, базовый для всех животных, наследуется от интерфейса IAlive и реализует свойство int Food
Herbo и Predator - наследники класса Animal для разделения животных на хищных и травоядных, оба класса реализую поля Animal (HealthLevel, Name, Food). Herbo дополнительно имеет поле KindLevel.

Thing - абстрактный класс, базовый для всех вещей, которые могут быть добавлены в инвентарь зоопарка, реализует IInventory (свойство int Number)

ZooGarden - основной класс зоопарка, он управляет животными и инвентарем. Ассоциирует интерфейсы для IZooReporter (для отчетов) и IVeterinary (для проверки здоровья животных)
Veterinary - ветеринарная клиника, для проверки здоровья животных. Реализует IVeterinary с методом bool IsHealthy().

Реализован класс для репортов в консоль о состоянии ZooGarden - реализует IZooReporter c 4 методами для вывода информации о разных сферах зоопарка.

В целом проект соблюдает SOLID принципы:
-S: каждый класс отвечает только за одну задачу.
-O: новые виды животных можно добавлять через наследования без изменения базовых классов. Легко расширяется система отчетности через IZooReporter.
-L: классы наследники могут заменять классы родителей и ничего не должно сломаться.
-I: IInventory, IVeterinary, IZooReporter, IAlive - четко разделяют функциональность.
-D: ZooGarden - зависит от абстракций, а не от конкретных реализаций.

Также в проекте использован DI контейнер.
